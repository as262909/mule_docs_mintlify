## Base URL

```
https://anypoint.mulesoft.com/hybrid/api/v1
```

## Required Headers

Every request to the Runtime Manager API must include the following headers:

| Header | Description | Example |
|--------|-------------|---------|
| `Authorization` | Bearer token obtained from the Anypoint login or Connected App OAuth flow | `Bearer e5d7f8a0-b345-...` |
| `X-ANYPNT-ENV-ID` | The target environment ID | `aabb1122-ccdd-...` |
| `X-ANYPNT-ORG-ID` | The organization ID | `xxxx-yyyy-zzzz-...` |

```bash
# Header template for all requests
-H "Authorization: Bearer {access_token}" \
-H "X-ANYPNT-ENV-ID: {environmentId}" \
-H "X-ANYPNT-ORG-ID: {organizationId}"
```

<Note>
  The Runtime Manager API manages **on-premises** Mule runtimes (servers, server groups, and clusters) and the applications deployed to them. **CloudHub** application deployments use the separate CloudHub API (`https://anypoint.mulesoft.com/cloudhub/api`).
</Note>

## Clusters

### `POST` /clusters

Create a new cluster from existing servers.

<Accordion title="Request body">
  ```json
  {
    "name": "prod-cluster-01",
    "multicastEnabled": false,
    "servers": [
      { "serverId": 101, "serverIp": "10.0.1.15" },
      { "serverId": 102, "serverIp": "10.0.1.16" }
    ]
  }
  ```
</Accordion>

<Accordion title="Example request">
  ```bash
  curl -X POST "https://anypoint.mulesoft.com/hybrid/api/v1/clusters" \
    -H "Authorization: Bearer {access_token}" \
    -H "X-ANYPNT-ENV-ID: {environmentId}" \
    -H "X-ANYPNT-ORG-ID: {organizationId}" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "prod-cluster-01",
      "multicastEnabled": false,
      "servers": [
        { "serverId": 101, "serverIp": "10.0.1.15" },
        { "serverId": 102, "serverIp": "10.0.1.16" }
      ]
    }'
  ```
</Accordion>

<Accordion title="Example response">
  ```json
  {
    "data": {
      "id": 50,
      "name": "prod-cluster-01",
      "multicastEnabled": false,
      "status": "RUNNING",
      "servers": [
        { "id": 101, "name": "mule-server-prod-01", "status": "RUNNING" },
        { "id": 102, "name": "mule-server-prod-02", "status": "RUNNING" }
      ]
    }
  }
  ```
</Accordion>

<Tip>
  Set `multicastEnabled` to `false` for unicast clusters (recommended for cloud/VPC environments) and provide each server's IP. Set it to `true` only if your network supports multicast discovery.
</Tip>

### `POST` /applications

Deploy a new application to a server, server group, or cluster.

<Accordion title="Request body (multipart/form-data)">
  | Field | Type | Required | Description |
  |---|---|---|---|
  | `targetId` | string | Yes | The ID of the server, server group, or cluster |
  | `artifactName` | string | Yes | The application name |
  | `file` | binary | Yes | The Mule application JAR file |
  | `configuration` | JSON string | No | Application properties as a JSON object |
</Accordion>

<Accordion title="Example request">
  ```bash
  curl -X POST "https://anypoint.mulesoft.com/hybrid/api/v1/applications" \
    -H "Authorization: Bearer {access_token}" \
    -H "X-ANYPNT-ENV-ID: {environmentId}" \
    -H "X-ANYPNT-ORG-ID: {organizationId}" \
    -F "targetId=50" \
    -F "artifactName=order-process-app" \
    -F "file=@/path/to/order-process-app-1.0.0-mule-application.jar"
  ```
</Accordion>

<Accordion title="Example response">
  ```json
  {
    "data": {
      "id": 3001,
      "name": "order-process-app",
      "status": "DEPLOYING",
      "desiredStatus": "STARTED",
      "target": {
        "id": 50,
        "name": "prod-cluster-01",
        "type": "CLUSTER"
      }
    }
  }
  ```
</Accordion>
